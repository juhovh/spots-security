This is where the open source development of the security related code for SPOTs will take place. All available code can accessed by clicking on the Subversion link on the left. The Sun SPOT SDK already supports code authentication using digital signatures. This code is available inside the DesktopSigning module. We've also developed code to implement SSL/TLS for encrypted communication between SPOTs and between SPOTs and clients/servers on the Internet. This code is now available as a technology preview but isn't included by default within the SPOT SDK. Please refer to the readme.txt file inside the SSL module for instructions on experimenting with the SSL/TLS code.

Roadmap

The SSL/TLS code offers secure communication capability over streams (e.g. radiostream or TCP). We are looking for help from our community to add DTLS (i.e. a TLS derivative called Datagram TLS, see RFC 4347) support which would extend the secure communication capability to radiograms and UDP. To join one of the spots-libraries projects you must be a member of both java.net and sunspotworld.com. Before requesting a membership role please first post a note introducing yourself on the Sun SPOT Projects forum on www.sunspotworld.com. Like most open source projects people wanting write access to the svn repository first need to demonstrate that they understand the current code and the project's coding practices. This is generally done through participation on the project's mailing list and the SunSpotWorld Forums, including reporting bugs and posting patches.
